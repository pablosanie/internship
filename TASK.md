# ТЗ для дипломного проекта

## Проект
**Многоканальный анализатор цифровых данных (UART + SPI)**

---

# 1. Назначение

Разработать устройство на базе STM32, предназначенное для:

- приёма цифровых данных по интерфейсам UART и SPI;
- буферизации входящих потоков;
- анализа структуры и параметров данных;
- предоставления пользователю информации о состоянии и характеристиках потоков.

---

# 2. Общая функциональность

Устройство должно:

1. Работать одновременно с двумя источниками данных:
   - UART
   - SPI

2. Обрабатывать входящие данные в реальном времени.

3. Предоставлять пользователю информацию через интерфейс управления.

---

# 3. Интерфейсы

## UART используется для:

1. Приёма данных.
2. Передачи результатов.
3. Ввода команд управления.

## SPI

1. Используется как дополнительный канал приёма данных.
2. Устройство должно работать в режиме Slave.

---

# 4. Основные функции

## 4.1 Приём данных

- Приём данных должен быть непрерывным.
- Не допускается потеря данных при нормальной нагрузке.
- Должна поддерживаться одновременная работа UART и SPI.

## 4.2 Буферизация

- Входящие данные должны накапливаться во внутренних буферах.
- Система должна корректно обрабатывать переполнение буфера.
- Поведение при переполнении должно быть определено и описано.

## 4.3 Анализ данных

Система должна выполнять анализ входящих потоков.

### Обязательные функции

- Подсчёт количества принятых байт (отдельно по каждому интерфейсу).
- Определение текущей скорости передачи данных.

### Дополнительные функции (не менее одной)

- Выделение пакетов по заданному признаку.
- Поиск заданной последовательности байт.
- Определение пауз между передачами (timeout).

## 4.4 Управление системой

Устройство должно поддерживать управление через текстовый интерфейс.

### Минимальные команды

- Запуск и остановка приёма данных.
- Включение/выключение отдельных интерфейсов.
- Получение текущего состояния системы.
- Сброс статистики.

## 4.5 Индикация и вывод информации

Система должна предоставлять:

- состояние (работает / остановлена);
- количество принятых данных;
- скорость передачи;
- информацию об ошибках.

---

# 5. Требования к архитектуре

Программное обеспечение должно быть:

1. Модульным.
2. Расширяемым.
3. Разделённым на уровни:
   - уровень работы с интерфейсами;
   - уровень обработки данных;
   - уровень взаимодействия с пользователем.

---

# 6. Требования к работе в реальном времени

- Система должна обрабатывать поток данных без заметных задержек.
- Приём данных не должен блокировать выполнение других задач.
- Должна быть обеспечена устойчивость при высокой нагрузке.

---

# 7. Обработка ошибок

Система должна учитывать и обрабатывать:

- переполнение буфера;
- ошибки передачи;
- некорректные команды пользователя.

---

# 8. Тестирование

Необходимо продемонстрировать:

- работу каждого интерфейса отдельно;
- одновременную работу UART и SPI;
- корректную обработку потока данных;
- реакцию системы на ошибки.

---

# 9. Результаты проекта

В рамках проекта должны быть представлены:

1. Рабочее устройство.
2. Исходный код.
3. Описание архитектуры.
4. Инструкция по использованию.
5. Демонстрация работы.

---

# 10. Критерии оценки

## Минимальный уровень

- Система принимает данные хотя бы по одному интерфейсу.
- Реализована базовая обработка.

## Средний уровень

- Оба интерфейса работают.
- Корректная буферизация и анализ.

## Высокий уровень

- Стабильная параллельная работа.
- Продуманная архитектура.
- Обработка ошибок.
- Удобный интерфейс управления.

## 2. Параметры

### 2.1 UART

1. Скорость: 57600 или 115200.
2. Формат: 8N1 или 8N2 (желательно).

### 2.2 SPI (Slave)

1. Режим: Mode 0.
2. Размер слова: 8 bit.
3. Частота: 100–500 кГц.

## 3. Формат данных

    0xAA 0x55 ... 0x01

## 4. Условия тестирования

- длинный поток без пауз;
- быстрые короткие пакеты;
- переполнение буфера.

## 5. Итоговые параметры

### UART

- 57600 или 115200 бод;
- 8N1.

### SPI

- Mode 0;
- 8 бит;
- 100–500 кГц.

### Поток

- пакетный;
- с паузами.
